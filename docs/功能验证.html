0. 环境准备与启动

0.1 依赖

本机用全局 py -3（Python 3.11），无 venv。如换机器或重装，先装依赖：

# 项目根目录 D:\CodeProject\Tools\YunShan
py -3 -m pip install -r requirements.txt
cd frontend
npm install
cd ..

0.2 启动（需两个终端，均在项目根目录）

# 终端 1：后端 API
py -3 -m uvicorn backend.main:app --reload

# 终端 2：前端
cd frontend
npm run dev
验证：
  • 终端 1 出现 Uvicorn running on http://127.0.0.1:8000，无报错堆栈。
  • 终端 2 出现 Local: http://localhost:5173/。
  • 浏览器打开 http://localhost:5173，顶部出现深色导航栏：YunShan 量化 / 策略回测 / 截面选股 / 个股详情 / 历史记录。
注意：行情源是国内站点，代码已自动让其直连、绕开本地代理（如 Clash 7890）。但仍需能正常访问东方财富/新浪行情。若拉数据报 502 / ProxyError，多为本机网络/代理问题，参见 第 10 节。

2. 策略回测（向量化引擎） 需联网

页面：策略回测（首页）。

步骤操作
1左侧默认：股票代码 000001，区间 2022-01-01 ~ 2023-12-31，策略 ma_cross，引擎"向量化（快）"。
2点 运行回测。
验证（预期结果）：
  • 顶部出现 000001 平安银行 与"查看个股详情 →"链接。
  • 8 张指标卡：年化收益率、超额收益、最大回撤、夏普、Sortino、Calmar、交易胜率、交易笔数；每张带 🟢/🟡/🔴/⚪ 评级徽章和一句白话点评。
  • 数值合理性：最大回撤为负数；年化是百分数；夏普量级一般在 -3~3；交易笔数为正整数。
  • "价格+信号"图：K 线/收盘价上叠加持仓区间。
  • "权益曲线"图：策略净值（蓝实线）与 买入持有基准（灰虚线）两条对比。
  • 鼠标悬停指标卡的"?"显示详细解释。
判定通过：四类视图（指标卡 / 价格信号 / 权益曲线 / 基准对比）齐全且数值合理；切换策略（如 rsi_reversal）后参数输入框随之变化，再次运行结果不同。

3. 事件驱动引擎 + 止损止盈 需联网

步骤操作
1左侧"回测引擎"切换为 事件驱动（支持止损止盈），会多出"止损 %""止盈 %"输入框。
2先留空止损止盈，运行一次，记下年化/回撤。
3再填 止损 8、止盈 20（单位 %），运行。
验证：
  • 两次都能正常出图出指标，无报错。
  • 加了止损后，结果与不加止损不同（通常最大回撤变浅 / 交易笔数变化），说明止损逻辑生效。
  • （一致性原理）事件驱动在无成本无止损时与向量化结果一致——这点由自动化测试 test_event_engine.py 保证（见第 8 节），手工无需复现。

4. 参数寻优 + 稳健性检验 需联网

步骤操作
1选策略 ma_cross（两个参数，便于看热力图），选"寻优目标指标"如 sharpe_ratio。
2点 网格寻优。
3点 走动检验。
验证 — 网格寻优：
  • 出现结果表（每行一组参数 + 各指标），按目标指标降序。
  • 因 ma_cross 恰好 2 个参数，表上方出现 双参数热力图（颜色映射目标指标）。
  • 下方出现 稳健性检验 面板：尝试组合数、观测 Sharpe、随机期望最高 Sharpe、Deflated Sharpe（%）、蒙特卡洛 p 值，每项带白话徽章（"很可能是真实 alpha""可能只是寻优运气""显著优于随机"等），并列出最优参数。
验证 — 走动检验：
  • 结果表每行一个 split，含样本内指标 is_* 与样本外 oos_*，可对比是否过拟合（样本内好、样本外崩）。
  • 走动检验模式下不显示热力图/稳健性面板（仅网格模式有）。

5. 截面多因子选股 需联网

页面：截面选股。

步骤操作
1"股票池"选 沪深300（000300）（建议先用指数，全市场较慢）。
2中部"选择因子与权重"：默认已勾 市盈率 和 60日动量。可勾更多，每个可改权重。基本面因子带"基本面"绿标，技术因子带"技术"蓝标。
3Top N 填 30。若勾了技术因子会多出"回看天数"输入（逐股拉历史，慢）。
4点 开始选股。
验证：
  • 出现蓝色 进度条 并显示百分比（后台任务 + 每秒轮询）。仅基本面因子时几乎秒完；含技术因子会逐步推进。
  • 完成后出现结果表：rank / code / name / score + 各因子原始值与标准化分 *_z。
  • 按 score 降序；点某行 code 链接跳转到该股 个股详情 页。
  • （方向正确性）只勾"市盈率"（低估值优）时，排名靠前的标的 PE 普遍较低。
提示：基本面因子（PE/PB/市值/换手/动量）来自一次全市场快照，快。技术因子需逐股拉价格，沪深300 全跑约数十秒到数分钟，取决于网络与缓存。

6. 个股详情（报价 / 分时 / 注意点） 需联网

页面：个股详情（也可从回测页/选股页跳入）。

步骤操作
1左侧输入代码（如 000001），点 查看（或回车）。
验证：
  • 顶部显示 代码 + 名称 + 最新价与涨跌幅（A股配色：红涨绿跌）。
  • 注意点 卡：标签化显示当前信号，如"RSI 超卖/超买""MACD 金叉/死叉""多头/空头排列""估值偏高""ROE…""主力净流入/流出"，按好/警示/差着色；每个标签带"技术/基本面/资金"分类。无触发时显示"暂无…"。
  • 分时图：当日价格曲线，附昨收虚线。（盘后/非交易时段可能为空，属正常，见限制）
  • 基本面 表：PE(TTM)/PB/PS/股息率/总市值/ROE/毛利率/营收增速/净利增速，取不到的显示"—"。
  • 另类数据 表：主力净流入(元)/占比%/北向持股市值。
  • 报价自动刷新：交易时段内停留 60 秒，最新价会自动更新（轮询，非推送）。

9. 纯接口验证（可选，用 curl / 浏览器） 需联网

不想点页面时，可直接打后端接口（PowerShell 用 curl.exe）：

# 列策略
curl http://127.0.0.1:8000/strategies

# 跑一次回测
curl -X POST http://127.0.0.1:8000/backtest -H "Content-Type: application/json" ^
  -d "{\"symbol\":\"000001\",\"start\":\"2022-01-01\",\"end\":\"2023-12-31\",\"strategy\":\"ma_cross\",\"params\":{\"short_window\":5,\"long_window\":20}}"

# 可选因子目录 / 股票池
curl http://127.0.0.1:8000/screening/factors
curl http://127.0.0.1:8000/universe

# 提交选股（返回 task_id），再轮询
curl -X POST http://127.0.0.1:8000/screening -H "Content-Type: application/json" ^
  -d "{\"universe_key\":\"index:000300\",\"factors\":[{\"key\":\"pe\",\"weight\":1}],\"top_n\":20,\"max_symbols\":300,\"lookback_days\":180}"
curl http://127.0.0.1:8000/screening/<task_id>

# 个股报价 / 信号 / 历史记录
curl http://127.0.0.1:8000/quotes/000001
curl http://127.0.0.1:8000/symbols/000001/signals
curl http://127.0.0.1:8000/experiments
验证：
  • 各接口返回 JSON、HTTP 200；/backtest 的 stats 含 sharpe_ratio，series 各数组等长。
  • /screening 先返回 {"task_id":...}，轮询到 status:"done" 后带 results。

10. 已知限制与排错

现象原因 / 处理
拉数据 502 / ProxyError / 连接被拒本机网络或代理问题。代码已对行情源直连，但仍需能访问东方财富/新浪。若开了 Clash 等代理，确认其在运行；或在 quant/config.py 确认 IGNORE_PROXY=True。
个股详情"分时图"为空非交易时段 / 盘后 akshare 可能无当日分时，属正常。交易日盘中再看。
基本面/另类数据某些字段是"—"akshare 对应接口未取到或列名改版。适配层做了防御式降级（取不到填 NaN→显示"—"），不影响其它功能。
北向持股、龙虎榜等数据缺失这类接口稳定性较弱，仅作展示标签、不参与回测；缺失不影响回测/选股主流程。
截面选股很慢勾了技术因子会逐股拉历史。改用指数/行业池、减小"最多扫描股票数"、或只用基本面因子。当天拉过的数据会缓存，二次更快。
前端报"无法加载策略列表（后端启动了吗？）"后端没起 / 端口不是 8000 / vite 代理被本机 HTTP 代理拦截。先确认终端 1 在跑。
数据缓存位置data/ 目录（gitignore）：行情 *_qfq.parquet+.meta.json（按天失效）、股票池/快照缓存、experiments.db。需要重置可删除该目录内容。
本手册对应 Phase 0–8 全部实现。设计与实施记录见 docs/DESIGN.md，运行说明见 README.md。
